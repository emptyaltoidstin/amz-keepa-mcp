## 📖 使用指南

### 标准流程详解

#### 步骤1: 采集Keepa数据

系统自动从Keepa API获取：
- 产品基础信息 (标题、品牌、类目)
- 所有变体ASIN
- 完整尺寸重量 (packageLength/Width/Height/Weight)
- 价格历史、销量排名
- 评论数据、退货率

#### 步骤2: 计算真实费用

基于Keepa数据自动计算：

```python
# FBA费用计算 (2026年费率)
计费重量 = max(实际重量, 体积重量)
体积重量 = (长×宽×高) / 166  (inch³)

# 佣金比例 (基于类目)
Electronics: 8%
Clothing: 17%
Jewelry: 20%
其他: 15%
```

#### 步骤3: 生成统一精算师报告

生成一份统一报告 (`{ASIN}_UNIFIED_ACTUARY.html`)：

**报告结构：**
```
├─ 交互式成本计算器
│  └─ 填入1688采购价 → 自动计算完整COGS
│
├─ 汇总指标面板
│  └─ 月度总利润 / 总销量 / 平均利润率 / ROI
│
├─ 投资建议
│  └─ GO / 谨慎 / 避免 + 置信度
│
├─ 帕累托分析
│  └─ 80/20核心变体识别
│
├─ 变体利润分析表
│  └─ 每个变体的详细利润数据
│
├─ 风险评估
│  └─ 风险项列表 + 等级
│
├─ 行动计划
│  └─ 30-60-90天优化建议
│
└─ 计算公式参考
   └─ 完整财务模型说明
```

#### 步骤4: 填入采购价 (唯一需要手动的步骤)

```
1. 打开1688.com
2. 使用"以图搜货"找到产品

4. 在交互式报告中填入
5. 系统自动计算完整COGS和利润

```
售价 (Amazon)

  - 退货成本

  ─────────────────

## ❓ 常见问题

### Q: 为什么只需要输入ASIN就能生成完整报告?

**A:** 系统会自动：
1. 从Keepa API获取产品数据 (尺寸、重量、价格等)
2. 自动发现所有变体
3. 基于真实尺寸计算FBA费用
4. 基于类目确定佣金比例
5. 生成包含163指标的完整报告

**唯一需要手动的**: 填入从1688找到的采购价 (因为需要人工确认供应商)

### Q: FBA费用准确吗?

**A:** 是的，基于：
- Keepa API提供的产品尺寸和重量
- 2026年最新FBA费率表
- 自动计算体积重量 vs 实际重量

### Q: 佣金比例是如何确定的?

**A:** 基于Keepa API提供的类目信息：

| 类目 | 佣金 |
|------|------|
| Electronics | 8% |
| Clothing | 17% |
| Jewelry | 20% |
| 其他 | 15% |

### Q: 如何获取1688采购价?

**A:** 
1. 打开 https://www.1688.com
2. 点击搜索框右侧"相机"图标 (以图搜货)
3. 粘贴Amazon产品图片
4. 找到相似产品，记录价格

### Q: 什么是TACOS?

**A:** TACOS (Total ACOS) = 广告总花费 / 总销售额

- 比ACOS更能反映广告对整体业务的影响
- 默认假设15%，可根据实际调整
