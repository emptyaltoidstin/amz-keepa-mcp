  • Sales volume allocation algorithm: weight = 1 / BSR^0.5
  • TACOS calculation: ad_cost = (revenue × 15%) / ad_orders
  • Mixed profit margins: blended = organic×pct + ad×pct
  • BSR-sales mapping: Piecewise power law function

Data processing capabilities:
  • Number of variants analyzed in a single run: Unlimited (Automatic batching)
  • Completeness of indicator collection: 163/163 (100%)
  • Report generation time: < 10 seconds
  • API call optimization: Batch query + automatic delay

Accuracy:
  • boughtInPastMonth data: 90%+
  • BSR estimation: 60-80%
  • Profit calculation: ±5% (Based on real COGS)

  Q: What to do about API current limit?
  A: It is normal. The system will wait automatically. Please wait patiently.

  Q: Sales estimates have large deviations?
  A: Calibrate the model and adjust the BSR with your own product-Sales volume mapping parameters

  Q: How to get COGS?
  A: Request price from supplier: product price + first leg + tariff

  Q: Order source ratio?
  A: Seller Central → Advertising → Measurement & Analytics
